# 历史上的今天 · 定时生成与邮件发送全流程

> **一句话**：用 **GitHub Actions** 按北京时间定时跑脚本，在仓库里生成（或覆盖）次日 `history-*.md`，再通过 **SMTP** 把正文发到邮箱；本地用同一套脚本联调，**环境变量**即「接口契约」。

---

## 1. 背景与目标

| 项 | 说明 |
|----|------|
| 要做什么 | 每日固定结构的历史稿 + 可选邮件推送 |
| 不依赖什么 | 不调用大模型 API，成本可控 |
| 时间基准 | **北京时间**：生成「次日」稿件；定时用 UTC cron 换算 |

---

## 2. 架构速览

```
GitHub 定时 / 手动触发
    → checkout → npm ci
    → generate-history-daily.ts（联网拉取“历史上的今天”数据，按固定模板编排并写入）
    → git commit & push（把 md 写回仓库）
    → send-history-mail.ts（读目标 md → nodemailer → SMTP）
```

| 组件 | 路径 / 说明 |
|------|----------------|
| 定时与工作流 | `.github/workflows/history-daily-mail.yml` |
| 生成逻辑 | `scripts/generate-history-daily.ts`（`lunar-javascript` 农历 / 节气） |
| 发送逻辑 | `scripts/send-history-mail.ts`（`nodemailer`） |
| 产出目录 | `src/assets/history/history-YYYY-MM-DD.md` |
| 运维说明（简版） | `docs/history-mail-setup.md` |

`package.json` 脚本：

| 命令 | 作用 |
|------|------|
| `npm run history:generate` | 生成目标日期的 md |
| `npm run history:send-mail` | 发送该日期对应文件为邮件正文 |
| `npm run history:daily` | 先生成再发送（串联） |

---

## 3. 定时任务怎么配（GitHub）

1. 在仓库 **Settings → Secrets and variables → Actions** 中配置下文「环境变量 / Secrets」表中的项（SMTP 使用邮箱服务商提供的 **授权码**，不是登录密码）。
2. 工作流已包含 **schedule** 与 **workflow_dispatch**（可手动跑一次）。
3. **Cron**：工作流里为 `30 13 * * *`（**UTC**），对应 **北京时间 21:30**。

生成规则要点：

- 默认按「**北京当天日期 + 1 天**」作为目标文件名；也可用 **`TARGET_DATE=YYYY-MM-DD`** 覆盖（本地或在 Actions 手动运行时填写 `target_date`）。
- **每次生成都会按脚本内固定结构重写目标文件**；若需本地保留手改、跳过已存在文件，设置 **`SKIP_EXISTING_HISTORY_FILE=true`**。

### 3.1 手动执行指定日期（GitHub 页面）

1. 打开仓库 → **Actions** → 选择 **Generate and Mail Daily History**。  
2. 点击右上角 **Run workflow**。  
3. 在输入框 **`target_date`** 填写目标日（格式：`YYYY-MM-DD`，例如 `2026-04-20`）。  
4. 选择分支（通常 `main`）后点击 **Run workflow**。  

说明：

- `target_date` **不填**：走默认逻辑（北京时间今天 + 1 天）。  
- `target_date` **填写**：生成并发送该日期对应的 `history-YYYY-MM-DD.md`。  
- 该日期文件若已存在，会按当前固定结构**覆盖重写**。

## 4. 稿件结构（约定）

生成结果由脚本拼装，核心区块顺序固定，包括：公历 / 农历行、可选「今日节日」、古代印记、近现代·国际、科技与互联网、中国近现代、国际要闻、今日出生、今日逝世、程序员视角，以及文末统一收尾句。

节日逻辑简述：

- 固定公历节日来自脚本内映射表；
- **节气**由 `lunar-javascript` 按目标日计算，命中则进入「今日节日」。

**节气与日历日**：节气对应的是太阳黄经到达某一刻的**交节时刻**（常精确到秒），万年历上标在哪一天，以**北京时间下该时刻落在哪一天**为准；不宜写「某日前后都算当天节气」这类模糊表述。

## 5. 环境变量（接口契约）

### 5.1 发信必填（CI 与本地一致）

| 变量 | 说明 |
|------|------|
| `SMTP_HOST` | 如 QQ：`smtp.qq.com` |
| `SMTP_PORT` | 常见 `465`（465 走 SSL） |
| `SMTP_USER` | 发件账号 |
| `SMTP_PASS` | SMTP **授权码** |
| `HISTORY_MAIL_TO` | 收件人 |

可选：`HISTORY_MAIL_FROM`（缺省则用 `SMTP_USER`）。


![Actions secrets and variables 配置](../images/blog/timedGenerationAndEmailSending/image.png)


### 5.2 调试常用

| 变量 | 说明 |
|------|------|
| `TARGET_DATE` | `YYYY-MM-DD`，指定要生成 / 发送哪一天对应的 `history-*.md` |
| `SKIP_EXISTING_HISTORY_FILE` | 设为 `true` 时，若目标 md 已存在则**不覆盖**（仅生成步骤生效） |

### 5.3 本地从文件读入配置

`send-history-mail.ts` 启动时会合并读取项目根目录 **`.env`** 与 **`.env.local`**（后者覆盖同名键；**已存在于进程环境中的变量不会被文件覆盖**）。敏感项建议只放 **`.env.local`**（仓库已忽略 `.env*`）。

## 6. 本地一键命令（PowerShell 示例）

```powershell
# 指定日期：生成
$env:TARGET_DATE = '2026-04-20'
npm run history:generate

# 同一日期：发邮件（需已配置 SMTP 等变量或 .env.local）
$env:TARGET_DATE = '2026-04-20'
npm run history:send-mail
```

Bash 可写为：`TARGET_DATE=2026-04-20 npm run history:daily`。

## 7. 排错与演进（按时间线归纳）

### 7.1 构建失败：`Unterminated regular expression literal`

**现象**：`vue-tsc` 报 `scripts/send-history-mail.ts` 某行正则未闭合。

**原因**：误写为 `text.split(/\n')`，正则起始 `/` 与字符串 `'` 混用。

**处理**：改为 `text.split('\n')`。

---

### 7.2 本地发信报「缺少 SMTP_HOST」

**原因**：仅依赖进程环境变量，而本机未 export、`.env` 里也未配邮件项。

**处理**：使用 **`.env.local`** 写入 SMTP 相关变量，或先在 shell 中设置再执行 `npm run history:send-mail`。

---

### 7.3 文末收尾句出现两次

**原因**：旧稿在最后一个 `##` 区块后仍带有 `---` 与收尾句；`parseSections` 把收尾句算进「程序员视角」正文，`buildMarkdown` 又在全文末尾追加同一句，生成时**重复**。

**处理**（已实现于生成脚本）：

- 抽取章节时从段尾剥掉空行、`---`、以及与全文统一的收尾句（可循环剥除）；
- 全文末尾保留**一处**收尾句，并与手工习惯一致：段前保留 `---` 与空行。

---

### 7.4 节气日期与万年历不一致

**说明**：以天文交节时刻换算到北京日期为准；若文稿日期不是「节气日」，不应把该节气写进当日节日。
