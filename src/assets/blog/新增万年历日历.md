# 新增万年历日历


---

## 1. 需求概览（你看到的效果）

- 点击页面头部时间（右侧时间显示）  
  → 弹出万年历面板（遮罩 + 卡片）
- 面板支持：
  - 月视图日历（前后月补齐）
  - 年/月可输入跳转
  - 上一月 / 下一月 / 今天 / 关闭
  - 选中日期详情（农历、干支、宜忌）
  - 节气展示（如 **谷雨**）
  - 节假日规则：**休 / 班**（法定放假、调休上班）
  - 倒计时：以“今天”为基准智能提示（距离节日/日期还有几天）
- 图标统一：全站使用 **Iconify**（`@iconify/vue`）

## 2. 核心依赖（用了什么）

### 2.1 `lunar-typescript`

用途：

- 公历 `Solar` ↔ 农历 `Lunar`
- 节气：`lunar.getJieQi()`
- 宜/忌：`lunar.getDayYi()` / `lunar.getDayJi()`
- 节日（公历/农历）：`solar.getFestivals()` / `lunar.getFestivals()`
- 法定节假日/调休：`HolidayUtil.getHoliday('YYYY-MM-DD')`

依赖安装：

```bash
npm install lunar-typescript
```

### 2.2 Iconify

用途：

- 统一全站图标渲染（避免伪元素/字符图标）
- 本功能用到：`mdi:calendar-clock-outline`

## 4. 日历数据是怎么生成的（关键逻辑）

### 4.1 42 格月视图（前后月补齐）

- 当前视图年月：`viewYear / viewMonth`
- 计算当月 1 号是周几
- 向前补齐上一月天数，向后补齐下一月天数
- 最终保证：
  - 总格子数为 42（6 行 × 7 列）
  - 每格都对应一个 `Solar`

### 4.2 “节气 / 节日 / 休 / 班”规则（最终版）

本功能中我们把“**显示文案**”和“**休班规则**”拆开，避免互相污染：

- **节气/节日文案**
  - 节气：`lunar.getJieQi()`
  - 节日：`solar.getFestivals()` / `lunar.getFestivals()`（只在真正节日当天显示）
- **休/班规则（法定）**
  - `HolidayUtil.getHoliday('YYYY-MM-DD')`
    - `isWork() = false` → 显示 `休`
    - `isWork() = true` → 显示 `班`
- **周末红色规则**
  - 周六/周日默认红（双休日）
  - 若当天是调休上班（`班`），则取消红色
  - 若当天是法定休假（`休`），保持红色并显示 `休`

### 4.3 倒计时文案（以今天为基准）

倒计时行遵循：

- 选中今天：提示 **距离下一个传统节日还有 X 天**
- 选中未来/过去：提示 **距离该日期/该节日还有/已过去 X 天**

其中“下一个传统节日”通过向后扫描若干天查找“节日（不含节气）”得到。

## 7. 快速排障（常见问题）

- **节日不显示**
  - 检查是否在“非节日当天”误显示（本实现已修复：不扩散到假期其它日期）
- **周末红色错位**
  - 检查周索引是否按周一为起点换算（本实现为周一=0）
- **调休上班日显示为红色**
  - 应以 `HolidayUtil` 的 `isWork()` 为准取消红色
